# ESP32-P4 Welder Control - Flashing Guide\n\n## Prerequisites\n\n### Hardware Requirements\n- **Waveshare ESP32-P4-WiFi6-Touch-LCD-7B** board\n- **USB-C Cable** (for power and data)\n- **Computer** with ESP-IDF installed\n\n### Software Requirements\n- ESP-IDF installed and configured\n- Project built successfully (`idf.py build` completed)\n- Drivers for USB-to-UART (usually auto-installed)\n\n---\n\n## Step 1: Identify the COM Port\n\n### **On Windows:**\n```bash\nwmic logicaldisk get name\nmode\n```\nOr check Device Manager under \"Ports (COM & LPT)\"\n\nLook for: `COM3`, `COM4`, `COM5`, etc.\n\n### **On Linux:**\n```bash\nls /dev/tty*\n```\nLook for: `/dev/ttyUSB0` or `/dev/ttyACM0`\n\n### **On macOS:**\n```bash\nls /dev/tty.*\n```\nLook for: `/dev/tty.usbserial-*` or `/dev/tty.SLAB_USBtoUART`\n\n---\n\n## Step 2: Connect Your Board\n\n1. **Connect USB-C Cable:**\n   - Plug USB-C into the ESP32-P4 board\n   - Connect other end to your computer\n   - Wait 2-3 seconds for device recognition\n\n2. **Verify Connection:**\n   ```bash\n   # Linux/macOS\n   ls -la /dev/tty*\n   \n   # Windows (PowerShell)\n   Get-PnpDevice -Class Ports\n   ```\n\n---\n\n## Step 3: Flash the Firmware\n\n### **Method 1: Using idf.py (Recommended)**\n\nNavigate to your project directory and run:\n\n```bash\n# Replace COM3 with your actual port\nidf.py -p COM3 flash monitor\n```\n\n**For Linux/macOS:**\n```bash\nidf.py -p /dev/ttyUSB0 flash monitor\n```\n\n**Parameters:**\n- `-p COM3` = Serial port (replace with your actual port)\n- `flash` = Write firmware to ESP32\n- `monitor` = Open serial monitor to view logs\n\n### **Method 2: Flash Only (Without Monitor)**\n\n```bash\nidf.py -p COM3 flash\n```\n\n### **Method 3: Flash with Specific Baud Rate**\n\n```bash\nidf.py -p COM3 -b 921600 flash monitor\n```\n\nCommon baud rates: `115200`, `230400`, `460800`, `921600`\n\n---\n\n## Step 4: Monitor Output\n\nOnce flashing completes, you should see:\n\n```\nStarting Welder Pulse Control System\nInitializing ADC for potentiometer\nInitializing DAC for voltage output\nInitializing PWM for pulse control\nInitializing BLE server\nInitializing LCD display\nWelder control system initialized successfully\n```\n\n### **Exit Monitor:**\n- Press `Ctrl + ]` (Windows/Linux)\n- Press `Ctrl + Cmd + Z` (macOS)\n\n---\n\n## Troubleshooting\n\n### Problem: \"Failed to connect to ESP32\"\n\n**Solution:**\n1. Check USB cable is properly connected\n2. Try different USB port on computer\n3. Verify correct COM port: `idf.py port-monitor COM?`\n4. Update USB drivers\n\n### Problem: \"Timed out waiting for packet header\"\n\n**Solution:**\n```bash\n# Hold BOOT button while flashing\nidf.py -p COM3 flash\n# Or press BOOT + RESET on the board\n```\n\n### Problem: \"Write timeout\"\n\n**Solution:**\n```bash\n# Try lower baud rate\nidf.py -p COM3 -b 115200 flash monitor\n```\n\n### Problem: \"Device not found\"\n\n**Solution:**\n- Check device in Device Manager (Windows) or `ls /dev/tty*` (Linux)\n- Install CH340/CP210x drivers if needed\n- Try different USB port\n\n---\n\n## Erase Flash Memory (If Needed)\n\n```bash\n# Erase entire flash\nidf.py -p COM3 erase-flash\n\n# Then reflash\nidf.py -p COM3 flash monitor\n```\n\n---\n\n## Commands Summary\n\n| Command | Purpose |\n|---------|----------|\n| `idf.py -p COM3 flash` | Flash firmware |\n| `idf.py -p COM3 monitor` | Open serial monitor |\n| `idf.py -p COM3 flash monitor` | Flash + monitor (recommended) |\n| `idf.py -p COM3 erase-flash` | Erase all flash memory |\n| `idf.py -p COM3 menuconfig` | Configure project settings |\n| `idf.py build` | Build project (no flash) |\n| `idf.py clean` | Remove build artifacts |\n\n---\n\n## Testing After Flash\n\n1. **Check Serial Output:**\n   - Watch for initialization messages\n   - Verify \"Welder control system initialized successfully\"\n\n2. **Test Potentiometer:**\n   - Connect 5kΩ potentiometer to GPIO35\n   - Adjust pot and check ADC readings in monitor\n\n3. **Test BLE:**\n   - Download BLE app on your phone (\"nRF Connect\" recommended)\n   - Scan for \"Welder_Control\" device\n   - Verify connection\n\n4. **Test DAC Output:**\n   - Connect multimeter to GPIO17/18\n   - Verify voltage changes with control input\n\n---\n\n## Advanced Flashing Options\n\n### Flash with Custom Partitions\n```bash\nidf.py -p COM3 partition-table-flash\n```\n\n### Minimal Flash (Factory App Only)\n```bash\nesptool.py -p COM3 write_flash 0x10000 build/welder_control.bin\n```\n\n### Flash Multiple Regions\n```bash\nesptool.py -p COM3 write_flash \\\n  0x0 build/bootloader/bootloader.bin \\\n  0x8000 build/partition_table/partition-table.bin \\\n  0x10000 build/welder_control.bin\n```\n\n---\n\n## Need Help?\n\n- **ESP-IDF Documentation:** https://docs.espressif.com/projects/esp-idf/\n- **Waveshare Wiki:** https://www.waveshare.com/wiki/ESP32-P4-WiFi6-Touch-LCD-7B\n- **Espressif GitHub:** https://github.com/espressif/esp-idf\n"
